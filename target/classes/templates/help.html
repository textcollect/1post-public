Frequently Asked Questions

  1. What is the meaning behind the name of this site (1post)?
    Because user can only post and reply once a day (every 24 hours, to be exact), the name "1post" was given to this project site to make it clear even to first-time visitors what the difference with popular social media sites like Twitter.
  2. This site looks interesting, how do I register?
    This site is invite-only for now. When you receive an invite, you will be asked for your desired username and password. An account will then be created based on the details received. To be sure an email invite is really sent from us, make sure to check that the email address ends with "@generation.org".
  3. Why can I not attach images to my posts or replies?
    That is one of the features of this site (NOT a bug). The reason being: there are already countless other social networking sites that allow you to do so (reddit, facebook, twitter, etc.). Kindly refer to those services if you wish for such functionality. The main idea behind this site is to provide a place for users to pop-in, post/reply, pop-out and continue with their daily lives. Adding images would just make loading the site and reading it slower.
  4. How was this site made?
    • This site was made by applying quite a number of modifications to the basic template of codes provided throughout the Generation SG Full-Stack Developer bootcamp.
    • For frontend, HTML, CSS and Javascript were used along with the Thymeleaf template engine and the Bootstrap toolkit.
    • MySQL was used as the database.
    • Finally, the backend was coded in Java with the use of the Spring Boot MVC framework to bring everything together.
  5. What are some of the challenges encountered when making this site?
    • Having to handle most of the work was rather intense. Thankfully, the public availability and advancement of Large Language Models such as ChatGPT made some of the work (such as suggesting ideas to implement the various features of the site) easier.
    • There were a couple of instances when a small change in the code led to everything breaking apart. Thankfully the issues were identified and fixed.
    • There were some difficulties in the backend when we were trying to link the user ids to the user table. After spending some time to look into the specifics in details, we were able to successfully link everything together.
  6. What would future revisions look like?
    • Redo the frontend to make use of the ReactJS library to make things smoother and more reactive.
    • Add a "delete" button for the user's own posts for easy deletion.
    • Edit functionality for user to edit their posts (This feature will only be available to verified-checkmarked users who subscribe to 1post-verified)
    • Open up registration to everyone once the server is able to handle the user demand (if said user demand exists).
    • (Tentative) Personal page for each user to view all of their posts.
